# 银行 ATM 自动取款系统配置与使用说明

本文档面向第一次下载本项目的使用者。仓库不包含任何真实邮箱、授权码或个人配置，使用者需要在自己的电脑上配置 SMTP 环境变量。

## 1. 环境要求

| 项目 | 要求 |
| --- | --- |
| Python | 推荐 3.10 或 3.11，项目只使用标准库 |
| 数据库 | SQLite，随 Python 标准库提供 |
| 网络 | 本机 TCP Socket，默认 `127.0.0.1:8888` |
| GUI | Tkinter，Windows 官方 Python 通常自带 |

检查 Python：

```powershell
python --version
```

## 2. 初始化数据库

进入项目根目录后执行：

```powershell
python database/init_db.py
```

默认账号：

| 用户名 | 密码 | 初始余额 | 默认邮箱 |
| --- | --- | ---: | --- |
| `zhangsan` | `123456` | `5000.00 RMB` | `demo@example.com` |
| `lisi` | `123456` | `3000.00 RMB` | `demo@example.com` |
| `admin` | `123456` | `10000.00 RMB` | `demo@example.com` |

如果要把默认账号邮箱改成自己的邮箱，请先设置 `ATM_DEMO_EMAIL`，再执行初始化数据库。

## 3. 配置邮件功能

邮件提醒不是主流程的前置条件。未配置 SMTP 时，登录、查询、取款、存款、流水和 RTT 测试仍然可以正常演示。

QQ 邮箱 PowerShell 示例：

```powershell
$env:ATM_DEMO_EMAIL="your@qq.com"
$env:ATM_SMTP_HOST="smtp.qq.com"
$env:ATM_SMTP_PORT="465"
$env:ATM_SMTP_USER="your@qq.com"
$env:ATM_SMTP_PASSWORD="your_smtp_authorization_code"
$env:ATM_SMTP_FROM="your@qq.com"
$env:ATM_SMTP_SSL="true"
python database/init_db.py
```

持久写入 Windows 用户环境变量的示例：

```powershell
[Environment]::SetEnvironmentVariable("ATM_DEMO_EMAIL", "your@qq.com", "User")
[Environment]::SetEnvironmentVariable("ATM_SMTP_HOST", "smtp.qq.com", "User")
[Environment]::SetEnvironmentVariable("ATM_SMTP_PORT", "465", "User")
[Environment]::SetEnvironmentVariable("ATM_SMTP_USER", "your@qq.com", "User")
[Environment]::SetEnvironmentVariable("ATM_SMTP_PASSWORD", "your_smtp_authorization_code", "User")
[Environment]::SetEnvironmentVariable("ATM_SMTP_FROM", "your@qq.com", "User")
[Environment]::SetEnvironmentVariable("ATM_SMTP_SSL", "true", "User")
```

检查 SMTP 配置：

```powershell
python -m email_service.smtp_email --check
```

发送测试邮件：

```powershell
python -m email_service.smtp_email --to your@qq.com
```

说明：

- `ATM_SMTP_PASSWORD` 应填写邮箱 SMTP 授权码，不是网页登录密码。
- 授权码不要写入源码，不要提交到 GitHub。
- 取款和存款成功后，服务器会后台尝试发送邮件，界面通常显示 `Email Notice: Queued`。
- 即使 SMTP 服务失败，也不会影响余额更新和交易流水。

## 4. 启动服务器

```powershell
python server/bank_server.py
```

正常输出：

```text
[INFO] Bank server started on 127.0.0.1:8888
[INFO] Waiting for ATM client connections...
```

如果端口被占用：

```powershell
python server/bank_server.py --host 127.0.0.1 --port 8899
```

## 5. 启动图形化客户端

新开一个终端：

```powershell
python client/atm_client.py
```

该命令默认打开中文图形化界面。操作顺序：

1. 点击 `连接服务器`。
2. 点击 `登录`，默认账号为 `zhangsan / 123456`。
3. 点击 `查询余额`。
4. 金额输入 `100`，点击 `取款`。
5. 金额输入 `50`，点击 `存款`。
6. 点击 `查询流水`。
7. 点击 `RTT 测试`。
8. 点击 `退出`。

命令行备用客户端：

```powershell
python client/atm_client.py --cli
```

## 6. 通信协议

客户端请求：

```text
HELO username
PASS password
BALA
WITHDRAW amount
DEPOSIT amount
FLOW
PING
QUIT
```

服务器普通响应格式：

```text
StatusCode StatusPhrase Data
```

RTT 测试中，客户端发送 `PING`，服务器直接返回 `PONG`。

## 7. 数据库重置

Windows PowerShell：

```powershell
Remove-Item database/atm.db
python database/init_db.py
```

macOS 或 Linux：

```bash
rm -f database/atm.db
python database/init_db.py
```

如果已经设置 `ATM_DEMO_EMAIL`，重置后默认账号邮箱会写入该邮箱；否则使用 `demo@example.com`。

## 9. 常见问题

客户端连不上服务器：

- 先确认服务器已经启动。
- 默认端口为 `8888`。
- 图形化客户端可以修改端口。

邮件未收到：

- 先执行 `python -m email_service.smtp_email --check`。
- 再执行 `python -m email_service.smtp_email --to your@qq.com`。
- 如果测试邮件失败，通常是 SMTP 服务未开启、授权码错误或当前网络阻断 SMTP。ATM 主流程不依赖邮件。

图形界面打不开：

- 确认 Python 安装包含 Tkinter。
- 也可以使用 `python client/atm_client.py --cli` 进入命令行备用界面。
