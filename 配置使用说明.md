# 银行 ATM 自动取款系统配置与使用文档

本文档对应当前最终整合作品，说明本地环境、启动顺序、客户端使用、SMTP 邮件配置、RTT 测试、数据库重置、验收测试和常见问题。项目已经整合成员 1 至成员 4 的实现内容，包括客户端界面、Socket 通信、中央服务器、SQLite 数据库、交易流水、SMTP 邮件提醒和 RTT 线路测试记录。

## 1. 项目内容

当前作品包含以下模块：

| 模块 | 文件位置 | 作用 |
| --- | --- | --- |
| 客户端命令行界面 | `client/atm_client.py`、`client/ui.py` | 启动 ATM 客户端，提供登录、查询、取款、存款、流水查询、RTT 测试菜单 |
| 客户端图形界面 | `client/gui_client.py` | Tkinter 图形化客户端，功能与命令行客户端一致 |
| 通信模块 | `client/communication.py` | 使用 TCP Socket 封装协议请求、解析响应、统计 RTT |
| 通信适配层 | `client/communication_adapter.py` | 让客户端界面加载通信模块 |
| 银行服务器 | `server/bank_server.py` | 监听 TCP 端口，接收并处理 ATM 客户端请求 |
| 命令处理 | `server/command_handler.py`、`server/response.py` | 解析协议命令，维护登录状态，生成统一响应报文 |
| 数据库与业务 | `database/account_service.py`、`database/database.py`、`database/init_db.py` | 初始化 SQLite 数据库，完成账户验证、余额更新、流水记录、RTT 记录 |
| 邮件服务 | `email_service/smtp_email.py` | 存款和取款成功后发送 SMTP 邮件提醒 |
| 测试用例 | `test/` | 覆盖客户端、通信、服务器、数据库和邮件异常处理 |

## 2. 运行环境

推荐环境如下：

| 项目 | 要求 |
| --- | --- |
| 操作系统 | Windows 10/11 |
| 终端 | PowerShell、CMD、Windows Terminal 或 VS Code Terminal |
| Python | 推荐 Python 3.10 或 Python 3.11；当前代码仅使用标准库 |
| 数据库 | SQLite，Python 标准库自带 `sqlite3` |
| 网络 | 本机 TCP Socket，默认地址 `127.0.0.1:8888` |
| 编码 | UTF-8 |
| 第三方依赖 | 无 |

检查 Python 版本：

```powershell
python --version
```

如果电脑上安装了多个 Python 版本，建议使用 Python 3.10 或 3.11 运行。项目在 Python 3.13 环境下也可以通过当前测试，但若教师严格按文档环境验收，优先使用 3.10 或 3.11。

## 4. 第一次运行配置

进入项目根目录后，先初始化数据库：

```powershell
python database\init_db.py
```

正常输出示例：

```text
Database initialized: E:\计算机网络\大作业\code\database\atm.db
```

说明：

- `database/atm.db` 是 SQLite 数据库文件。
- 如果数据库文件不存在，程序会自动创建。
- 初始化会创建账户表、交易流水表、RTT 记录表，并写入默认测试账号。

项目不需要安装第三方依赖。`requirements.txt` 仅说明当前项目使用 Python 标准库。

## 5. 启动服务器

在项目根目录执行：

```powershell
python server\bank_server.py
```

正常输出示例：

```text
[INFO] Bank server started on 127.0.0.1:8888
[INFO] Waiting for ATM client connections...
```

服务器默认监听：

```text
127.0.0.1:8888
```

如果端口 `8888` 被占用，可以指定其他端口：

```powershell
python server\bank_server.py --host 127.0.0.1 --port 8899
```

使用自定义端口时，图形化客户端可以在界面中填写对应端口；命令行客户端默认连接 `8888`，演示时建议直接使用默认端口。

停止服务器：

```text
在服务器终端按 Ctrl+C
```

## 6. 启动命令行客户端

服务器启动后，新开一个 PowerShell 或 CMD 终端，进入同一个项目根目录：

```powershell
cd E:\计算机网络\大作业\code
python client\atm_client.py
```

命令行客户端启动后会显示菜单：

```text
1. Login
2. Balance Query
3. Withdraw
4. Deposit
5. Transaction Flow Query
6. Network RTT Test
7. Logout / Exit
```

推荐演示流程：

1. 选择 `1. Login`，输入用户名和密码。
2. 选择 `2. Balance Query`，查询当前余额。
3. 选择 `3. Withdraw`，输入取款金额，例如 `100`。
4. 选择 `4. Deposit`，输入存款金额，例如 `50`。
5. 选择 `5. Transaction Flow Query`，查看余额查询、取款、存款和流水查询记录。
6. 选择 `6. Network RTT Test`，进行 5 次 RTT 测试并查看最小值、最大值、平均值。
7. 选择 `7. Logout / Exit`，退出客户端。

## 7. 启动图形化客户端

项目额外提供 Tkinter 图形化界面。服务器启动后，在项目根目录执行：

```powershell
python client\gui_client.py
```

图形化客户端使用步骤：

1. 确认 `Host` 为 `127.0.0.1`，`Port` 为 `8888`。
2. 点击 `Connect` 连接服务器。
3. 输入用户名和密码，点击 `Login`。
4. 点击 `Balance` 查询余额。
5. 在 `Amount` 中输入金额，点击 `Withdraw` 或 `Deposit`。
6. 点击 `Flow` 查询交易流水。
7. 点击 `RTT Test` 进行通信链路往返时延测试。
8. 点击 `Quit` 退出。

图形化客户端和命令行客户端连接的是同一个服务器和同一个数据库，不要同时用两个客户端操作同一个账户进行演示，避免流水顺序不易说明。

## 9. 通信协议说明

客户端请求格式：

```text
COMMAND [data]\n
```

服务器响应格式：

```text
StatusCode StatusPhrase Data\n
```

编码和结束符：

- 编码：UTF-8。
- 每条请求和响应以换行符 `\n` 结尾。
- 服务器按行读取请求，处理后返回一行响应。

支持命令如下：

| 命令 | 客户端报文示例 | 说明 |
| --- | --- | --- |
| `HELO` | `HELO zhangsan` | 发送用户名 |
| `PASS` | `PASS 123456` | 发送密码并完成登录 |
| `BALA` | `BALA` | 查询账户余额 |
| `WITHDRAW` | `WITHDRAW 100.00` | 取款 |
| `DEPOSIT` | `DEPOSIT 50.00` | 存款 |
| `FLOW` | `FLOW` | 查询交易流水 |
| `PING` | `PING` | RTT 测试 |
| `QUIT` | `QUIT` | 退出连接 |

常见服务器响应：

| 响应示例 | 含义 |
| --- | --- |
| `200 OK HELO accepted, User: zhangsan` | 用户名存在，等待密码 |
| `200 OK Login success` | 登录成功 |
| `201 OK Balance: 5000.00 RMB` | 余额查询成功 |
| `202 OK Withdraw success, Balance: 4900.00 RMB, Email Notice: Sent/Failed` | 取款成功，并给出邮件发送结果 |
| `203 OK Deposit success, Balance: 4950.00 RMB, Email Notice: Sent/Failed` | 存款成功，并给出邮件发送结果 |
| `204 OK Flow query success: ...` | 流水查询成功 |
| `300 OK Quit success` | 退出成功 |
| `400 ERROR Wrong password` | 密码错误 |
| `401 ERROR User not found` | 用户不存在 |
| `402 ERROR Insufficient balance` | 余额不足 |
| `403 ERROR Invalid amount` | 金额非法 |
| `404 ERROR Please login first` | 未登录就执行受保护操作 |
| `500 ERROR Server error` | 非法命令或服务器异常 |

RTT 测试中，客户端发送 `PING`，服务器直接返回：

```text
PONG
```

客户端只把 `PONG` 视为有效 RTT 响应。

## 10. SMTP 邮件配置

系统在取款和存款成功后，会尝试向账户邮箱发送提醒邮件。邮件使用 SMTP 环境变量配置。

QQ 邮箱示例：

```powershell
$env:ATM_SMTP_HOST="smtp.qq.com"
$env:ATM_SMTP_PORT="465"
$env:ATM_SMTP_USER="your@qq.com"
$env:ATM_SMTP_PASSWORD="smtp_authorization_code"
$env:ATM_SMTP_FROM="your@qq.com"
$env:ATM_SMTP_SSL="true"
```

变量说明：

| 环境变量 | 是否必须 | 示例 | 说明 |
| --- | --- | --- | --- |
| `ATM_SMTP_HOST` | 是 | `smtp.qq.com` | SMTP 服务器地址 |
| `ATM_SMTP_PORT` | 是 | `465` | SMTP 端口 |
| `ATM_SMTP_USER` | 是 | `your@qq.com` | SMTP 登录账号 |
| `ATM_SMTP_PASSWORD` | 是 | `xxxxxxxx` | SMTP 授权码，不是网页登录密码 |
| `ATM_SMTP_FROM` | 是 | `your@qq.com` | 发件人邮箱 |
| `ATM_SMTP_SSL` | 否 | `true` | 是否使用 SSL，默认 `true` |

检查 SMTP 配置：

```powershell
python -m email_service.smtp_email --check
```

发送真实测试邮件：

```powershell
python -m email_service.smtp_email --to your@qq.com
```

注意：

- 如果不配置 SMTP，取款和存款仍会成功，但结果中会显示 `Email Notice: Failed`。
- 邮件发送失败不会回滚数据库操作，余额和交易流水仍然正常更新。
- 演示真实邮件发送前，需要提前在邮箱网页端开启 SMTP 服务并获取授权码。
- 不要把真实授权码提交到 GitHub；只在本机 PowerShell 中临时设置环境变量。

## 11. RTT 记录与导出

客户端执行 RTT 测试后，结果会写入 SQLite 表 `rtt_record`，用于通信链路质量统计分析。

在客户端中执行：

```text
6. Network RTT Test
```

程序会显示：

```text
RTT 1 : xx.xx ms
RTT 2 : xx.xx ms
RTT 3 : xx.xx ms
RTT 4 : xx.xx ms
RTT 5 : xx.xx ms
Min RTT : xx.xx ms
Max RTT : xx.xx ms
Avg RTT : xx.xx ms
Status  : 网络状态优秀/良好/一般/较差
Record  : RTT database record #1
```

导出 RTT 记录为 CSV：

```powershell
python -c "from database.account_service import export_rtt_records_csv; export_rtt_records_csv('rtt_records.csv')"
```

导出后会在项目根目录生成：

```text
rtt_records.csv
```

该文件可用于截图或统计分析。

## 13. 数据库重置

演示取款、存款、流水查询和 RTT 测试后，数据库内容会发生变化。若需要恢复初始状态，推荐删除数据库后重新初始化：

```powershell
Remove-Item database\atm.db
python database\init_db.py
```

说明：

- 删除数据库后重新初始化，会恢复默认账号余额。
- 交易流水和 RTT 记录也会被清空。
- 如果只是执行 `python database\init_db.py`，程序会补建表和默认账号，但不会主动清空已有流水。

## 14. 现场演示建议顺序

正式演示前建议按以下顺序准备：

1. 打开 PowerShell，进入项目目录。
2. 执行 `python database\init_db.py` 初始化数据库。
3. 执行 `python server\bank_server.py` 启动服务器。
4. 新开一个终端，执行 `python client\atm_client.py` 启动命令行客户端。
5. 使用 `zhangsan / 123456` 登录。
6. 依次演示余额查询、取款、存款、流水查询。
7. 演示 RTT 测试，说明客户端和服务器之间进行了多次 `PING/PONG` 往返时延统计。
8. 如需演示邮件，提前设置 SMTP 环境变量，并在取款或存款成功后观察 `Email Notice: Sent`。
9. 如需演示图形界面，再启动 `python client\gui_client.py`。

## 15. 常见问题

### 15.1 客户端无法连接服务器

可能原因：

- 没有先启动服务器。
- 服务器端口不是 `8888`。
- 端口被其他程序占用。
- 防火墙拦截本地连接。

处理方法：

```powershell
python server\bank_server.py
```

确认服务器输出 `Waiting for ATM client connections...` 后，再启动客户端。

### 15.2 提示 Please login first

原因：未登录就执行了余额查询、取款、存款或流水查询。

处理方法：先选择 `1. Login`，输入正确用户名和密码。

### 15.3 提示 Invalid amount

原因：金额为空、不是数字、为 0 或负数。

处理方法：输入正数，例如：

```text
100
50.5
```

### 15.4 提示 Insufficient balance

原因：取款金额大于账户余额。

处理方法：先查询余额，再输入小于或等于余额的取款金额。

### 15.5 提示 Email Notice: Failed

原因：SMTP 未配置、授权码错误、网络异常或邮箱服务商拒绝连接。

处理方法：

1. 如果不演示邮件功能，可以忽略该提示。
2. 如果需要演示真实邮件，按第 10 节配置 SMTP 环境变量。
3. 运行 `python -m email_service.smtp_email --check` 检查缺失项。
4. 运行 `python -m email_service.smtp_email --to your@qq.com` 测试真实发送。

### 15.6 中文显示乱码

优先使用 Windows Terminal 或 VS Code Terminal。若仍乱码，可在 PowerShell 中执行：

```powershell
chcp 65001
```

然后重新运行客户端。

### 15.7 数据库被演示改动

演示取款、存款、查询流水和 RTT 测试都会写入数据库。需要恢复初始状态时执行：

```powershell
Remove-Item database\atm.db
python database\init_db.py
```
