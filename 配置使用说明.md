# 银行 ATM 自动取款系统配置与使用说明

本文档面向第一次拿到本仓库的使用者，目标是让别人克隆仓库后可以独立完成配置、启动、演示和故障排查。

## 1. 系统简介

本项目实现了一个银行 ATM 自动取款系统。客户端和服务器通过 TCP Socket 通信，服务器使用 SQLite 保存账户、余额、交易流水和 RTT 测试记录。系统在存款和取款成功后会尝试发送 SMTP 邮件提醒。

已实现功能：

- 用户登录。
- 余额查询。
- 取款。
- 存款。
- 交易流水查询。
- 取款和存款后的 SMTP 邮件提醒。
- 客户端与服务器通信链路 RTT 测试。
- RTT 测试结果保存到数据库并支持导出 CSV。
- 命令行客户端。
- Tkinter 图形化客户端。
- 自动化测试。

## 2. 运行环境

推荐环境：

| 项目 | 要求 |
| --- | --- |
| 操作系统 | Windows 10/11，macOS 或 Linux 均可 |
| Python | 推荐 Python 3.10 或 Python 3.11 |
| 数据库 | SQLite，Python 标准库内置 |
| 网络 | 本机 TCP Socket，默认 `127.0.0.1:8888` |
| 第三方依赖 | 无，当前项目只使用 Python 标准库 |

检查 Python：

```powershell
python --version
```

如果电脑上安装了多个 Python 版本，请确认执行 `python` 时使用的是 Python 3.10 或 3.11。项目当前不需要执行 `pip install`。

图形化客户端依赖 Python 自带的 `tkinter`。Windows 官方 Python 安装包通常已包含 `tkinter`；如果在精简 Linux 环境中缺少 `tkinter`，可以先使用命令行客户端完成演示。

## 8. 使用图形化客户端

服务器启动后，在项目根目录执行：

```powershell
python client/gui_client.py
```

使用步骤：

1. `Host` 填写 `127.0.0.1`。
2. `Port` 填写 `8888`。
3. 点击 `Connect`。
4. 输入用户名和密码，点击 `Login`。
5. 点击 `Balance` 查询余额。
6. 在 `Amount` 中输入金额，点击 `Withdraw` 或 `Deposit`。

8. 点击 `RTT Test` 测试通信链路。

命令行客户端和图形化客户端使用同一个服务器和同一个数据库。演示时建议只打开一个客户端，避免多窗口同时操作导致流水顺序不清晰。

## 9. 通信协议

客户端请求格式：

```text
COMMAND [data]\n
```

服务器响应格式：

```text
StatusCode StatusPhrase Data\n
```

编码：UTF-8。

支持命令：

| 命令 | 示例 | 说明 |
| --- | --- | --- |
| `HELO` | `HELO zhangsan` | 发送用户名 |
| `PASS` | `PASS 123456` | 发送密码并完成登录 |
| `BALA` | `BALA` | 查询余额 |
| `WITHDRAW` | `WITHDRAW 100.00` | 取款 |
| `DEPOSIT` | `DEPOSIT 50.00` | 存款 |
| `FLOW` | `FLOW` | 查询交易流水 |
| `PING` | `PING` | RTT 测试 |
| `QUIT` | `QUIT` | 退出连接 |

常见响应：

| 响应示例 | 含义 |
| --- | --- |
| `200 OK HELO accepted, User: zhangsan` | 用户名通过，等待密码 |
| `200 OK Login success` | 登录成功 |
| `201 OK Balance: 5000.00 RMB` | 余额查询成功 |
| `202 OK Withdraw success, Balance: 4900.00 RMB, Email Notice: Sent/Failed` | 取款成功 |
| `203 OK Deposit success, Balance: 4950.00 RMB, Email Notice: Sent/Failed` | 存款成功 |
| `204 OK Flow query success: ...` | 流水查询成功 |
| `300 OK Quit success` | 退出成功 |
| `400 ERROR Wrong password` | 密码错误 |
| `401 ERROR User not found` | 用户不存在 |
| `402 ERROR Insufficient balance` | 余额不足 |
| `403 ERROR Invalid amount` | 金额非法 |
| `404 ERROR Please login first` | 未登录就执行受保护操作 |
| `500 ERROR Server error` | 非法命令或服务器异常 |

RTT 测试时，客户端发送 `PING`，服务器直接返回：

```text
PONG
```

客户端只把 `PONG` 作为有效 RTT 响应。

## 10. SMTP 邮件配置

系统在取款和存款成功后会尝试发送邮件提醒。如果未配置 SMTP，取款和存款仍会成功，但结果中会显示：

```text
Email Notice: Failed
```

### 10.1 PowerShell 配置示例

以 QQ 邮箱为例：

```powershell
$env:ATM_SMTP_HOST="smtp.qq.com"
$env:ATM_SMTP_PORT="465"
$env:ATM_SMTP_USER="your@qq.com"
$env:ATM_SMTP_PASSWORD="smtp_authorization_code"
$env:ATM_SMTP_FROM="your@qq.com"
$env:ATM_SMTP_SSL="true"
```

### 10.2 macOS 或 Linux 配置示例

```bash
export ATM_SMTP_HOST="smtp.qq.com"
export ATM_SMTP_PORT="465"
export ATM_SMTP_USER="your@qq.com"
export ATM_SMTP_PASSWORD="smtp_authorization_code"
export ATM_SMTP_FROM="your@qq.com"
export ATM_SMTP_SSL="true"
```

### 10.3 配置项说明

| 环境变量 | 是否必须 | 说明 |
| --- | --- | --- |
| `ATM_SMTP_HOST` | 是 | SMTP 服务器地址 |
| `ATM_SMTP_PORT` | 是 | SMTP 端口，QQ 邮箱 SSL 通常为 `465` |
| `ATM_SMTP_USER` | 是 | SMTP 登录邮箱 |
| `ATM_SMTP_PASSWORD` | 是 | SMTP 授权码，不是网页登录密码 |
| `ATM_SMTP_FROM` | 是 | 发件人邮箱，通常与登录邮箱一致 |
| `ATM_SMTP_SSL` | 否 | 是否使用 SSL，默认 `true` |

检查当前 SMTP 配置：

```powershell
python -m email_service.smtp_email --check
```

发送真实测试邮件：

```powershell
python -m email_service.smtp_email --to your@qq.com
```

安全提醒：

- 不要把真实邮箱授权码写入源码。
- 不要把真实邮箱授权码提交到 GitHub。
- 如果只是演示 ATM 主流程，可以不配置 SMTP。

## 11. RTT 测试和导出

客户端执行 RTT 测试后，会向服务器发送至少 5 次 `PING`，统计每次 `PONG` 返回的耗时。

测试结果包括：

- 每次 RTT，单位毫秒。
- 最小 RTT。
- 最大 RTT。
- 平均 RTT。
- 网络状态。
- 数据库记录编号。

RTT 结果会保存到 SQLite 表 `rtt_record`。

导出 RTT 记录为 CSV：

```powershell
python -c "from database.account_service import export_rtt_records_csv; export_rtt_records_csv('rtt_records.csv')"
```

导出文件：

```text
rtt_records.csv
```

## 12. 数据库重置

取款、存款、流水查询和 RTT 测试都会写入数据库。需要恢复初始状态时，可以删除数据库文件后重新初始化。

Windows PowerShell：

```powershell
Remove-Item database/atm.db
python database/init_db.py
```

macOS 或 Linux：

```bash
rm -f database/atm.db
python database/init_db.py
```

重新初始化后，默认账号余额恢复为初始值，交易流水和 RTT 记录会清空。

## 15. 常见问题

### 15.1 客户端无法连接服务器

先确认服务器终端已经显示：

```text
[INFO] Waiting for ATM client connections...
```

如果没有，请先启动服务器：

```powershell
python server/bank_server.py
```

如果端口被占用，可以换端口启动服务器：

```powershell
python server/bank_server.py --port 8899
```

图形化客户端可以直接填写新端口。命令行客户端默认使用 `8888`，一般建议演示时保持默认端口。

### 15.2 提示 Please login first

原因：没有登录就执行了余额查询、取款、存款或流水查询。

处理：先登录，再执行其他操作。

### 15.3 提示 Invalid amount

原因：金额为空、不是数字、为 0 或负数。

处理：输入正数，例如 `100` 或 `50.5`。

### 15.4 提示 Insufficient balance

原因：取款金额大于账户余额。

处理：先查询余额，再输入小于或等于余额的取款金额。

### 15.5 提示 Email Notice: Failed

原因：SMTP 未配置、授权码错误、网络异常或邮箱服务商拒绝连接。

处理：

1. 不演示邮件功能时可以忽略。
2. 要演示邮件功能时，按第 10 节配置 SMTP。
3. 先运行 `python -m email_service.smtp_email --check` 检查配置。
4. 再运行 `python -m email_service.smtp_email --to your@qq.com` 发送测试邮件。

### 15.6 中文显示乱码

优先使用 Windows Terminal 或 VS Code Terminal。

PowerShell 中也可以先执行：

```powershell
chcp 65001
```

然后重新运行客户端。

### 15.7 图形化客户端打不开

可能原因：

- 当前 Python 环境没有 `tkinter`。
- 当前系统没有图形桌面环境。

处理：

- Windows 用户建议安装 Python 官方完整版本。
- 如果暂时无法使用 GUI，可直接使用命令行客户端 `python client/atm_client.py`。
