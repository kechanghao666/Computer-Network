# 银行 ATM 项目现场演示说明

本文档用于现场演示。仓库不包含真实邮箱或 SMTP 授权码；如果需要演示真实邮件，请在演示机器上先按 `配置使用说明.md` 配置环境变量。

## 1. 演示前准备

进入项目目录：

```powershell
cd Computer-Network
```

初始化数据库：

```powershell
python database/init_db.py
```

启动服务器：

```powershell
python server/bank_server.py
```

看到以下内容即可：

```text
[INFO] Bank server started on 127.0.0.1:8888
[INFO] Waiting for ATM client connections...
```

## 2. 启动客户端

新开一个 PowerShell：

```powershell
cd Computer-Network
python client/atm_client.py
```

这会打开中文图形化界面。命令行备用入口：

```powershell
python client/atm_client.py --cli
```

## 3. 推荐演示流程

1. 点击 `连接服务器`。
2. 点击 `登录`，默认账号为 `zhangsan / 123456`。
3. 点击 `查询余额`，说明服务器从 SQLite 查询余额。
4. 金额输入 `100`，点击 `取款`，说明服务器校验余额、扣款、记录流水，并后台尝试邮件提醒。
5. 金额输入 `50`，点击 `存款`，说明服务器增加余额、记录流水，并后台尝试邮件提醒。
6. 点击 `查询流水`，展示 `BALANCE_QUERY`、`WITHDRAW`、`DEPOSIT`、`FLOW_QUERY` 等记录。
7. 点击 `RTT 测试`，展示 5 次 `PING/PONG` 往返时延、最小值、最大值、平均值和网络状态。
8. 点击 `退出`。

## 4. 讲解要点

项目结构：

- `client`：图形化客户端、命令行备用界面、Socket 通信封装和 RTT 统计。
- `server`：银行中央服务器，监听 `127.0.0.1:8888`，解析协议命令并调度业务。
- `database`：SQLite 数据库，保存账号、余额、交易流水和 RTT 记录。
- `email_service`：SMTP 邮件模块，取款和存款成功后后台尝试发送提醒。
- `test`：自动化测试，覆盖客户端、通信、服务器、数据库和邮件异常处理。

通信协议：

```text
HELO username
PASS password
BALA
WITHDRAW amount
DEPOSIT amount
FLOW
PING
QUIT
```

普通响应格式：

```text
StatusCode StatusPhrase Data
```

RTT 测试：

- 客户端发送 `PING`。
- 服务器返回 `PONG`。
- 客户端统计多次往返耗时。

## 5. 邮件说明

如果没有配置 SMTP，或 SMTP 服务不可用，ATM 主流程仍然正常：

- 余额会更新。
- 流水会记录。
- RTT 测试不受影响。
- 邮件任务会后台尝试执行，界面可能显示 `Email Notice: Queued`。

真实邮件演示前请先运行：

```powershell
python -m email_service.smtp_email --check
python -m email_service.smtp_email --to your@qq.com
```

测试邮件能收到后，再演示取款和存款后的邮件提醒。

## 6. 演示数据重置

```powershell
Remove-Item database/atm.db
python database/init_db.py
```

重置后默认余额恢复，流水和 RTT 记录清空。
